# 方案 1：关节内置舵机

## 一句话结论

每个关节配一套"电机 + 减速齿轮 + 位置传感器 + 控制电路"，直接放在关节处，用高减速比把电机转速转化成大力矩。优点是控制简单、成本低、模块化易维护；缺点是指端惯量大、柔顺控制弱，且反驱差是减速器物理特性决定的，软件补不回来。

## 原理

每个舵机模块内部链路：

> RS-485 总线指令 → MCU（微控制器） → 电机驱动（H 桥） → 直流电机 → 行星减速器 → 输出轴

同时有两条反馈路径：电机轴上的 12-bit 磁编码器（4096 格/圈）把转速和位置报给 MCU；电流采样电路把电机电流报给 MCU（这是力矩估算的数据来源）。

MCU 里运行三级 PID 闭环：最外是位置环，中间是速度环，最内是电流环。"位置模式"下三级全用；"电流限位置模式"下位置环输出直接接电流上限约束，速度环被跳过——这是操作研究最常用的模式，设目标角度同时设最大电流作为软性力限制。

行星减速器的减速比（约 200–350:1）是整个方案的关键数字：电机原生力矩只有约 0.02 N·m，经过 300 倍放大后达到 1–4 N·m，满足手指关节所需；但代价是反向效率极低——从外面推关节需要克服 300 倍放大的齿轮摩擦才能让关节动起来，感觉像是锁死的。这是这个方案无法做好接触力控制的根本原因，是减速器的物理特性，不是软件可以解决的问题。

## 和其他方案的区别——关键在"电机放哪里"

人类手指其实是腱绳方案——控制手指弯曲的肌肉在前臂，通过肌腱拉动手指。关节内置方案反其道而行之，把每个"肌肉"直接塞进了每个关节旁。这一个决定，决定了系统几乎所有特性：

- 电机在关节处 → 手指里有几百克的金属 → 指端惯量大 → 高速运动难控制 → 碰到东西会用较大力撞上去
- 电机在掌部（腱绳方案）→ 手指只有轻巧的铰链和细线 → 指端极轻 → 动力学性能好 → 代价是腱绳摩擦带来的控制复杂度

气动方案（硅胶充气）走的是另一条路：不用电机，用气压推动手指弯曲，天然柔顺，但速度只有 1–5 Hz，做不了精细操作（详见 [方案 4：气动驱动](方案4-气动驱动.md)）。

## 为什么成为主流

两个字：够用。对于绝大多数操作研究（RL 抓取、模仿学习、遥操作）来说，需要的不是精细力控，而是可靠的位置跟踪 + 可复现的硬件 + 快速上手。关节内置舵机把工程复杂度降到了极低——任何会写 Python 的研究者，拿到 LEAP Hand，两天内就能跑起来第一个实验。这种低门槛让它成为全球实验室共享算法基准的标准平台。做不了高端化，但可以作为科研原型机、日常中低端机。

## 应用场景、限制与趋势

### 高自由度 → 腱绳（已成为共识）

低自由度的手倾向关节内置舵机，高自由度的手倾向腱绳方案。驱动因素不是"自由度数本身"，而是电机重量的空间累积效应：每加一个关节内置电机，就在手指某处增加 40–80g 质量。对 3 DOF 手指还能接受，对 4 DOF（加 DIP 关节）和 5 根手指，质量会在指端迅速堆积——指端电机（DIP/PIP 位置）质量 × 力臂² 带来大量指端惯量，手指越长、DOF 越高越显著，到某个程度手指重到运动控制质量无法接受。

正确的因果链是：**高 DOF → 更多关节需要电机 → 如果全部内置则指端很重 → 为了保持轻量，把电机移到远端（腱绳）**。不是 DOF 本身排斥关节内置，而是组件尺寸在物理上产生了这种推力——如果哪天做出 5g 输出 1 N·m 的微型电机，这个限制就消失。

LEAP Hand（16 DOF，关节内置）是反例：确实做到了较高 DOF，代价是每根手指显著偏重，动力学性能受影响，学术研究还能接受，部署场景就吃力了。

### 接触力精度 → 腱绳；位置精度 → 关节内置（已达成共识）

**接触力控精度**（灵巧操作真正需要的）：腱绳更好。原因：指端轻，碰到物体时冲击力小，接触过渡更稳定；拮抗腱可主动调节关节刚度，关节内置做不到；Capstan 减速比（2–20:1）比行星减速（200–350:1）的 backdrivability 好得多，能感知更小的外力。所以"高精度灵巧操作倾向于腱绳"准确说是接触力控精度，而非位置精度。

**位置精度**：关节内置更好。绝对编码器直接读电机轴，分辨率极高，无腱绳弹性和摩擦引入的误差；腱绳系统的 Bowden 管摩擦和弹性形变反而让位置估算不稳定。

### 与准直驱（QDD）的边界

TESOLLO DG-5F 使用直接驱动方案，20 DOF，重量约 880g，带触觉传感器，是"高 DOF + 直驱"的组合，既不是关节内置高减速比，也不是腱绳。这条路目前成本高、体积大，但打破了"高精度 = 必须腱绳"的判断。QDD 为何被选择、解决了什么问题、有哪些非 backdrivability 不可的任务场景，详见独立文档 [方案 3：准直驱 QDD](方案3-准直驱QDD.md)（原文中这部分内容混在本方案的"应用场景"里，现拆分出去以保持每个方案文档的聚焦）。
