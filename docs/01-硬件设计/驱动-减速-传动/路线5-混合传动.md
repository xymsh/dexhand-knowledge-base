# 路线5：混合传动

## 一句话结论

前四条技术路线（[路线1：关节内置驱动](路线1-关节内置驱动.md)、[路线2：腱绳驱动](路线2-腱绳驱动.md)、[路线3：连杆驱动](路线3-连杆驱动.md)、[路线4：流体驱动](路线4-流体驱动.md)）各自假设整只手用同一种传动机制。但不少真实产品会在同一只手上，跨机制组合出各部位职责分工——这条路线专门收纳"混合本身就是设计亮点"的方案。

## 分类边界：什么才算"混合传动"，什么不算

这是一条容易被滥用的路线，所以边界要划清楚：**只有当跨机制混合是这个产品的核心设计决策、而不是某个次要细节时，才归入这里。** 比如 Shadow Dexterous Hand 整体是[路线2：腱绳驱动](路线2-腱绳驱动.md)，只是指尖 DIP 关节用了个小连杆做被动耦合——这种局部细节继续按占主导地位的传动路径（腱绳驱动）归类，不需要挪到这里。真正该归入"混合传动"的，是像下面两个方案家族这样，"混合"本身就是整只手最核心的设计取舍。

## 两个方案家族

### 家族A：拇指独立驱动 + 其余四指欠驱动

**设计思路**：拇指用独立的电机（配合连杆或腱绳）实现对掌姿态调整，因为拇指对掌是抓握任务里最关键、最需要独立控制的自由度；其余四指则共享少数电机，通过欠驱动耦合联动，换取整体成本和控制复杂度的下降。这本质上是把"全驱动"和"欠驱动"两种驱动耦合策略，按手指分区分别应用，而不是整只手统一选择。

**代表产品**：这是商业假肢手领域很常见的设计思路（如 Ottobock、i-Limb、bebionic 等品牌的多款产品都采用类似的"拇指独立对掌+其余四指欠驱动"策略），但具体到某一款产品的机构细节（拇指用的是连杆还是腱绳、其余四指的欠驱动耦合方式）本文尚未逐一核实，先记录设计思路，代表产品和参数待补充。【建议核对】

### 家族B：刚柔混合

**设计思路**：刚性骨架/连杆机构负责手指的主体传动和结构强度，指尖/指腹局部引入软体材料或[路线4：流体驱动](路线4-流体驱动.md)（如气动辅助的柔性垫），在不牺牲整体结构刚度和位置精度的前提下，换取局部接触柔顺性和抓握安全性。这条思路呼应了[技术路线地图：驱动、减速、传动](00-技术路线地图.md)"分歧与共识"里提到的"刚性仿人手 vs 软体手"之争的一种折中方向：未来可能走向"刚性骨架 + 柔顺关节 + 软指腹 + 触觉传感 + 主动控制"的混合形态。

**代表产品**：目前没有能拍板确认机构细节的具体代表产品，属于本次重构分类体系时，为了让分类框架保持完整而先占的位——如果这条思路本身没有实际验证需要，也可以先删除这个方案家族，等真正找到合适的代表产品再补上。【待补充】
